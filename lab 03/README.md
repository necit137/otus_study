#### решение проблемы №1:

**1. Анализ масок подсети**

**Исходная маска (`255.255.255.0`)**

- В двоичном виде: `11111111.11111111.11111111.00000000`
- Биты сети: *24 (первые три октета)*
- Биты узлов: *8 (последний октет)*

**Новая маска (`255.255.255.224`)**

- В двоичном виде: `11111111.11111111.11111111.11100000`
- Биты сети: *27 (24 исходных + 3 новых)*
- Биты узлов: *5 (`32 - 27 = 5`)*

**2. Расчёт параметров**
   
  **а) Количество бит подсети (добавленных)**
  - Было: 24 бита сети
  - Стало: 27 бит сети
  - Добавилось: `27 - 24 = 3 бита подсети`
    
  ***Ответ: `3 бита подсети`***

  **б) количество созданных подсетей**
  - Добавили 3 бита → значит, можно создать `2³ = 8 подсетей`.
    
  ***Ответ: `8 подсетей`***

  **в) Количество бит узлов в подсети**
  - Всего бит в IPv4: 32
  - Биты сети: 27
  - Биты узлов: `32 - 27 = 5`

  ***Ответ: `5 бит узлов`***

  **г) Количество узлов в подсети**
  - На узлы выделено 5 бит → всего возможных адресов `2⁵ = 32`.
  - Но 2 адреса нельзя использовать:
    - Первый (`все нули`) — адрес сети.
    - Последний (`все единицы`) — широковещательный адрес.
  - Доступных узлов: `32 - 2 = 30`.
    
  ***Ответ: `30 узлов`***

**3. Определение текущей подсети**

IP-адрес: `192.168.200.139`

Новая маска: `255.255.255.224`

  **a)Находим сетевой адрес (AND между IP и маской)**
  - Переводим IP и маску в двоичный вид:
    `192.168.200.139` → `11000000.10101000.11001000.10001011`
    `255.255.255.224` → `11111111.11111111.11111111.11100000`
